# APP STATE: 宥めよ

このファイルは、会話ゲーム「宥めよ」固有の仕様、状態、状態遷移、変更履歴を記録する。

## 1. アプリ概要

- タイトル: `宥めよ`
- 種別: テキスト会話ゲーム
- 目的: 限られた発言回数の中で相手の信頼を得る。
- 核となる体験: 発言しなければ状況は動かないが、言葉選びを誤ると緊張度が上がる。

## 2. 現在の仕様

- プレイヤーは入力欄に一文を入力し、`送る` ボタンで発言する。
- 入力文に含まれる単語をルールベースで判定し、相手の反応と状態を更新する。
- 緊張度、信頼度、残り発言数を画面に表示する。
- 会話履歴を画面に表示する。
- 成功・失敗後は `もう一度` ボタンで初期状態に戻せる。

## 3. 管理する状態

| 状態 | 初期値 | 意味 |
| --- | --- | --- |
| `input` | 空文字 | プレイヤーが現在入力している一文 |
| `tension` | `2` | 相手の緊張度。最大 `5` |
| `trust` | `0` | 相手の信頼度。`4` 以上で成功 |
| `turnsLeft` | `5` | 残り発言数 |
| `reply` | `……もういい。全部、わかってるんだろ。` | 現在表示する相手の反応 |
| `history` | 空配列 | プレイヤーの発言と相手の反応の履歴 |
| `result` | `playing` | `playing`、`succeeded`、`failed` のいずれか |

## 4. 状態遷移

### 発言時

1. 入力文の前後空白を除く。
2. 入力が空、またはゲーム終了済みなら何もしない。
3. 入力文に含まれる単語を判定する。
4. 緊張度または信頼度を更新する。
5. 残り発言数を `1` 減らす。
6. 成功・失敗条件を順番に判定する。
7. 相手の反応と会話履歴を更新する。
8. 入力欄を空にする。

### 成功・失敗判定の優先順位

1. 緊張度が `5` 以上なら `failed`。
2. 信頼度が `4` 以上なら `succeeded`。
3. 残り発言数が `0` なら `failed`。
4. いずれにも当てはまらなければ `playing` を継続する。

### 再挑戦時

- すべての状態を初期値へ戻す。

## 5. 単語ルール

### 信頼度を上げる単語

| 単語 | 変化 | 相手の反応 |
| --- | --- | --- |
| `ごめん` | 信頼度 `+1` | `……謝れば済むと思ってるわけじゃ、ないよな。` |
| `大丈夫` | 信頼度 `+1` | `大丈夫って……どうして、そう言える。` |
| `聞かせて` | 信頼度 `+1` | `……聞いて、どうするつもりだ。` |

### 緊張度を上げる単語

| 単語 | 変化 | 相手の反応 |
| --- | --- | --- |
| `落ち着いて` | 緊張度 `+1` | `落ち着け？　簡単に言うなよ。` |
| `お前` | 緊張度 `+2` | `……その呼び方、やめろ。` |
| `知らない` | 緊張度 `+2` | `知らない？　じゃあ、どうしてここにいる。` |

### 判定対象外の入力

- 緊張度 `+1`
- 相手の反応: `……それで、何が言いたい。`
